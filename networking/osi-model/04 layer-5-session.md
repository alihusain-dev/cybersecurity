# layer 5 - session layer

## what is the session layer?

- creates and maintains connections between devices
- keeps communication active while data is being exchanged
- closes the connection when communication is finished

## how it works

- creates a session between two devices
- keeps the session active while data is transferred
- closes the session when it is no longer needed

## key terms

### session

- an active connection between two devices
- exists while communication is taking place

### checkpoints

- save points during communication
- if data is lost, communication can resume from the latest checkpoint instead of starting over

## examples

- logging into a website
- video calls
- online gaming sessions

## analogy

like a phone call.

- starting the call = creating a session
- talking = maintaining the session
- hanging up = closing the session

if the call drops, you reconnect and continue instead of restarting everything.

## key takeaway

- session layer = manages connections between devices
- creates, maintains, and closes sessions
- uses checkpoints to reduce the amount of data that must be resent
