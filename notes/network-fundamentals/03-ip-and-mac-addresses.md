# ip and mac addresses

## device identification

devices have two ways of being identified:

### ip address

- identifies where a device is on a network
- can change over time

**analogy:**
- home address

### mac address

- identifies the physical device
- usually permanent
- assigned by the manufacturer

**analogy:**
- fingerprint

## ipv4

- most common ip addressing system
- made up of 4 octets (0–255)

**example:**

- 192.168.1.1

## public vs private ip

### private ip

- used inside a local network
- not visible on the internet

**examples:**

- 192.168.1.77
- 192.168.1.74

### public ip

- identifies your network on the internet
- assigned by your isp
- multiple devices can share one public ip

**example:**

- 86.157.52.21

**analogy:**

- public ip = house address
- private ip = room number

## ipv6

- newer version of ip addressing
- solves ipv4 address shortage

**example:**

- 2a00:22c4:a531:c500:425f:cce6:c36b

## mac address

- media access control address
- unique identifier for a network interface

**example:**

- a4:c3:f0:85:ac:2d

**structure:**

- first half = manufacturer
- second half = unique device identifier

## spoofing

### mac spoofing

- changing a device's mac address

### ip spoofing

- faking the source ip address in packets

## key takeaway

- ip address = where
- mac address = who
- private ip = local network
- public ip = internet
- ipv6 replaces ipv4 as more addresses are needed
